# **Crate**

**Crate** (произносится как _«крейт»_, в переводе — «ящик» или «контейнер») — это **минимальная единица сборки и компиляции в Rust**.

Если проводить аналогию с другими языками программирования, то crate — это аналог **assembly** в .NET (C#) или **package/module** в Java/Go.

## 📦 Два вида крейтов

Крейт может состоять из множества модулей (файлов `.rs`), но на выходе компилятор `rustc` всегда собирает его в один из двух вариантов:

1. **Binary Crate (Бинарный крейт):**
    
    - Это полноценная программа, которую можно запустить (скомпилированный `.exe` файл на Windows).
        
    - Обязательно содержит функцию `fn main()`.
        
    - Главным файлом (точкой входа) обычно является `src/main.rs`.
        
2. **Library Crate (Библиотечный крейт):**
    
    - Это библиотека кода (аналог `.dll` на Windows), которую нельзя запустить саму по себе. Она пишется для того, чтобы её код использовали другие программы.
        
    - Не содержит `fn main()`.
        
    - Главным файлом является `src/lib.rs`.
        

## 🌐 Где живут чужие крейты?

В экосистеме Rust есть официальный центральный репозиторий (аналог NuGet для .NET, npm для Node.js или PyPI для Python). Он называется **[crates.io](https://crates.io/)**.

Когда вам нужно добавить в проект стороннюю библиотеку (например, для работы со случайными числами `rand` или для парсинга JSON `serde`), вы просто прописываете её имя и версию в вашем файле **`Cargo.toml`**:

Ini, TOML

```
[dependencies]
rand = "0.8.5" # Это внешняя зависимость (внешний crate)
```

Cargo сам пойдет на сайт _crates.io_, скачает этот крейт, скомпилирует его и подключит к вашему проекту.

## 🧩 Крейт vs Модуль (Module)

Их часто путают, но разница простая:

- **Crate** — это весь ваш проект целиком (сборка).
    
- **Module** — это просто отдельный файл или папка с кодом _внутри_ вашего крейта, используемый для наведения порядка (через ключевое слово `mod`).
