# Отличия типав
Это один из самых фундаментальных вопросов в Rust. Разница между ними сводится к тому, **где** эти строки хранятся в памяти и **кто** ими владеет.

Если коротко: `String` — это динамически изменяемая строка в куче (heap), а `str` — это неизменяемая строка «где-то еще» (внутри бинарника или в куче), на которую мы обычно смотрим через ссылку `&str`.

## 🏛️ Разбор на пальцах

### 1. `String` (Владеемая строка)

Это тип, который владеет своими данными. Он устроен точно так же, как `Vector` (динамический массив).

- **Где хранится:** Текст лежит в **куче (heap)**. Память выделяется динамически во время работы программы.
    
- **Мутабельность:** Ее можно изменять (дописывать символы, удалять, менять размер).
    
- **Управление памятью:** Когда переменная типа `String` выходит из области видимости, память в куче автоматически освобождается.
    

```Rust
// Создаем пустую String и дописываем в нее текст
let mut my_string = String::new();
my_string.push_str("Hello");
my_string.push('!'); // Теперь там "Hello!"
```

### 2. `str` и `&str` (Строковый срез / String Slice)

Сам по себе тип `str` в Rust называют _динамически измеряемым типом_ (DST). Компилятор не знает его точный размер на этапе компиляции, поэтому создать переменную типа `str` напрямую **нельзя** (код `let x: str = ...` не скомпилируется).

Вместо этого мы всегда используем **`&str`** (ссылку на строковый срез). Это «умный указатель», который состоит из двух вещей: адреса начала строки и её длины.

- **Где хранится:** Где угодно. Чаще всего — прямо внутри скомпилированного бинарника (если это строковый литерал), либо это ссылка на кусок существующей `String`.
    
- **Мутабельность:** Строго **неизменяемый** (read-only).
    
- **Владение:** `&str` не владеет данными, он просто «смотрит» на них.
    

```Rust
// Строковый литерал. Текст "Hello" зашит прямо в ваш .exe файл.
// Тип переменной s — это &'static str (живет вечно)
let s = "Hello"; 
```

## 🔄 Как они работают вместе (Аналогия)

Представьте, что **`String`** — это сундук с сокровищами, который вы можете расширять, докладывать туда вещи и носить с собой. А **`&str`** — это координатная записка: _"Сундук стоит там-то, длина содержимого — 5 метров"_. Вы можете дать эту записку кому угодно, читать данные можно, но расширить сундук через записку нельзя.

Вы можете легко сделать срез из `String`:


```Rust
let hello_string = String::from("Hello, World!");

// Берем ссылку на часть String. Тип s — это &str
let s: &str = &hello_string[0..5]; // "Hello"
```

## 🛠️ Что и когда использовать?

- Используйте **`String`**, если вам нужно владеть данными (например, передавать их в другие структуры), изменять строку на лету или если вы принимаете данные из сети/файла.
    
- Используйте **`&str`** в качестве аргументов функций. Это золотой стандарт Rust.

```Rust
// Плохо: функция принимает только String
fn greet_bad(name: String) {} 

// Идеально: функция принимает &str
// Сюда можно передать и литерал ("Sanya"), и ссылку на String (&my_string)
fn greet_good(name: &str) {} 
```

# Конвертация

Чем `String::from("Hello, World!")` отличается от `"Hello, World!"to_string()`?

Если коротко: **с технической точки зрения в современном Rust разницы нет вообще.** Обе команды делают абсолютно одно и то же — выделяют память в куче (heap), копируют туда текст `"Hello, World!"` и возвращают вам полноценный объект типа `String`.

Раньше между ними была крошечная разница в производительности, но сейчас они компилируются в идентичный машинный код. Выбор между ними — это исключительно вопрос вашего вкуса и стиля кода.

Тем не менее, под капотом они работают через разные механизмы языка:

## 🔬 Что происходит под капотом?

### 1. `String::from("...")`

Этот способ использует стандартный трейт (интерфейс) `From`.

- Конкретно для строк в стандартной библиотеке Rust прописано: _«Как создать `String` из типа `&str`»_.
    
- **Плюс:** Выглядит очень наглядно и явно. Сразу видно, какой тип данных мы создаем (`String`), передавая туда аргумент.
    

### 2. `"...".to_string()`

Этот способ использует трейт `ToString`.

- Этот трейт автоматически доступен для любого типа, который умеет красиво выводить себя на экран (реализует трейт `Display`).
    
- Внутри себя `to_string()` для строк просто вызывает тот самый `String::from()`.
    
- **Плюс:** Приятный ООП-стиль (вызов метода через точку). Его часто удобнее писать в цепочках методов (method chaining).
    

## ⏱️ Миф о производительности

В старых статьях по Rust (многолетней давности) можно встретить совет: _«Используйте `String::from`, потому что `to_string()` работает медленнее»_.

> ❌ **Это устаревшая информация.** Раньше `to_string()` делал лишние круги через механизм форматирования текста. Сейчас компилятор оптимизирует это "из коробки", и обе операции работают с одинаковой (максимальной) скоростью.

## 🛠️ Есть ли третий вариант?

Да, в современном коде на Rust очень часто используют третий, еще более короткий вариант:


```Rust
let s = "Hello, World!".to_owned();
```

Метод `.to_owned()` буквально означает _«сделай мне копию данных, которой я буду владеть»_. Для `&str` он точно так же создаст `String`. Многие разработчики предпочитают именно его, так как он семантически точнее передает суть операции в Rust — клонирование данных в единоличное владение.
